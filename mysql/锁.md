- RC
  - 快照读不加锁，走mvcc
  - 写需要加锁：行级排他锁
    - 若条件存在索引，则只锁住所修改的数据行
    - 若条件不存在索引，则innodb引擎会将整个表的每一行进行加锁。然后server层会将不满足条件的进行解锁

- RR

  - 快照读走mvcc
  - 当前读（加锁select、update、insert、delete）, 使用next-key锁。
    - 行锁和GAP锁形成Next-Key锁

- Serializable

  - 读加锁、写加锁、读写互斥、写写互斥
  - 对于insert采用Next-key锁

- 什么时候加Next-key锁

  | 场景                   | 是否加 Next-Key Lock |
  | ---------------------- | -------------------- |
  | 普通 SELECT            | ❌ 不加               |
  | 唯一索引等值（存在）   | ❌ 只加记录锁         |
  | 唯一索引等值（不存在） | ✅ 加 Gap Lock        |
  | 范围查询               | ✅ 加 Next-Key Lock   |
  | 非唯一索引等值         | ✅ 加 Next-Key Lock   |
  | UPDATE / DELETE 范围   | ✅ 加 Next-Key Lock   |

- Gap锁

  - 锁住的是“索引区间”，阻止其他事务在这个区间内 INSERT 新记录
  - 只阻止 INSERT不阻止 UPDATE 已存在的行

- 意向锁

  - 为什么需要意向锁？

    > 当事务A想要加表级锁时，它要判断这个表是否被其他事务加过锁了：先看这表是否被加了表级锁，若没有进一步判断是否加上了行级锁，这时就需要遍历整个表，效率很低。所以innodb引入了一种锁：意向锁

  - 意向锁机制：

    - 意向锁属于表锁

    - 意向锁分为意向共享锁(IS)和意向排他锁(IX)

    - 意向锁不直接锁数据，而是用来表明：表里是否存在行锁

    - 意向锁由innodb自动添加

      - 加行级共享锁

        ```sql
        SELECT * FROM t WHERE id = 1 LOCK IN SHARE MODE;
        ```

        Innodb会自动会给表t加`IS`,然后再给`id=1`这一行加`S`行锁

      - 加行级排他锁

        ```sql
        UPDATE t SET v = 1 WHERE id = 1;
        ```

        Innodb会自动会给表t加`IX`,然后再给`id=1`这一行加`X`行锁

    - 意向锁与意向锁之间不排斥、意向锁和表锁不排斥

    - 意向锁与表锁之间：只有共享表锁与意向共享锁兼容，其他的不是兼容的

      | 请求锁 \ 已持有锁  | **IS**（意向共享） | **IX**（意向排他） | **S**（表级共享） | **X**（表级排他） |
      | ------------------ | ------------------ | ------------------ | ----------------- | ----------------- |
      | **IS**（意向共享） | ✅                  | ✅                  | ✅                 | ❌                 |
      | **IX**（意向排他） | ✅                  | ✅                  | ❌                 | ❌                 |
      | **S**（表级共享）  | ✅                  | ❌                  | ✅                 | ❌                 |
      | **X**（表级排他）  | ❌                  | ❌                  | ❌                 | ❌                 |

      > 例如：事务 A 必须先申请该表的意向共享锁，成功后再申请数据行的行锁。事务 B 申请表锁时，数据库查看该表是否已经被其他事务加上了表级锁；如果发现该表上存在意向共享锁，说明表中某些数据行上存在共享锁，事务 B 申请的写锁会被阻塞。
